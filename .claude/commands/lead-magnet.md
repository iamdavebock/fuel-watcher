# Lead Magnet

Design and produce a complete lead magnet end-to-end: concept → spec → content → landing copy → delivery plan. Coordinates `leadgen` (design), `copywriter` (words), and `offer-architect` (naming + bridge to core offer).

---

## Inputs (gather first)

Ask all at once:

1. What's the core offer this magnet should lead to?
2. Who is the avatar? (specific buyer, not broad market)
3. What's the narrow problem we can solve *completely* for free?
4. What format suits the audience? (checklist / template / audit-scorecard / mini-course / swipe file / calculator — or "recommend one")
5. Where will traffic come from? (existing channel, content, ads, outreach)

If the user can't answer 1–3, stop and run `/offer-audit` or delegate to `brand-strategist` first — a magnet without a clear core offer leaks value.

---

## Build Sequence

1. **`leadgen`** — magnet spec: narrow problem, magnet type (reveal-problem / sample-solution / one-step-of-many), consumption time (<15 min to first value), the bigger problem it exposes, CTA inside
2. **`offer-architect`** — name it (MAGIC formula, 5 variants ranked) and design the bridge: what's the natural next step from magnet → core offer?
3. **`copywriter`** — produce: the magnet content itself (or outline if it needs Dave's expertise), landing page copy (3 headline + 2 CTA variants), 3-email delivery/follow-up sequence
4. **`leadgen`** — delivery plan: capture form fields (minimum viable), instant-delivery mechanism, qualification question (one max), metric targets (opt-in rate, magnet→conversation rate)

---

## Quality Bar

- [ ] Solves the narrow problem COMPLETELY — good enough to charge for
- [ ] First value in under 15 minutes of consumption
- [ ] Exposes the bigger problem the core offer solves (the bridge is explicit)
- [ ] One CTA inside the magnet — no link farms
- [ ] Named like a product, not a file ("The 6-Week Booking Blueprint" not "free guide")
- [ ] Capture form asks only for what will actually be used

---

## Output

```
lead-magnet/
├── spec.md            # leadgen's design spec + metric targets
├── content.md         # the magnet itself (or production outline)
├── landing-copy.md    # headlines, body, CTAs
├── email-sequence.md  # 3-email delivery + follow-up
└── delivery-plan.md   # capture, delivery, qualification, metrics
```

Finish by stating the ONE metric that decides if this magnet works (usually opt-in rate or magnet→booked-conversation rate) and the review date.
